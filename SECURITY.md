# Security policy

## Supported versions

Security fixes are provided for the latest published release. Source snapshots and older releases are not supported release channels.

## Report a vulnerability privately

Use [GitHub's private vulnerability report](https://github.com/Dkrynen/lac/security/advisories/new) so maintainers can investigate before details are public.

Do not open a public issue, discussion, or pull request for a suspected vulnerability. Include the affected version, operating system, reproduction steps, impact, and any safe proof of concept. Remove API keys, license keys, tokens, personal data, and unrelated machine details from the report.

Private vulnerability reporting is a mandatory public-launch control. If the private report form is unavailable, the release gate is incomplete; do not disclose the issue publicly while arranging a private contact path with the maintainer.

## Scope

This policy covers the MIT-licensed LAC Core repository and its released installers. Reports involving the separately distributed Local Pro add-on or the private LAC Cloud service can use the same private channel and should identify the affected product clearly.

Release provenance begins at the documented [supply-chain trust boundary](docs/security/SUPPLY-CHAIN-TRUST.md); it is not a claim that older public history was retroactively signed.

This project does not currently advertise a bug bounty or guaranteed response time. We will acknowledge valid reports and coordinate fixes and disclosure as capacity permits.
